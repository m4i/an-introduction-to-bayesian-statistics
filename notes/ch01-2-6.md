# 1.2.6 問題設定としての確率変数

## 3行サマリ

1. 二段構え: 確率空間 $(\Omega, \mathcal{F}, P)$ = 抽象的な「世界」/ 確率変数 $X$ = 具体的な問題を設定する装置
2. 同じ $\Omega$ 上に複数の確率変数を載せられる → 結合分布・条件つき確率の基盤
3. 注意1.8: 独立同分布な無限列の構成は厳密には非自明、入門者は事実として受け入れてOK

確率変数の **思想的な役割** を整理する節。なぜわざわざ「確率空間 + 確率変数」という二段構えで考えるのか。

## 二段構えの哲学

### 確率空間 $(\Omega, \mathcal{F}, P)$ — 抽象的な「世界」

「ありうるすべての確率世界」を抽象的に用意する。詳細には立ち入らない。

### 確率変数 $X$ — 具体的な問題を設定する装置

抽象世界から **興味の対象を取り出す写像**:
- コイン投げを考えたい → $X : \Omega \to \lbrace H, T\rbrace$
- サイコロ → $X : \Omega \to \lbrace 1, ..., 6\rbrace$
- $[0,1]$ 上の点 → $X : \Omega \to [0, 1]$

### 本文のコイン例は定義1.6（非実数値）の例

本書 1.2.6 のコイン例は、 $X : \Omega \to \lbrace H, T\rbrace$ と行き先を **非数値** で書いている。これは定義1.7（実数値）ではなく **定義1.6（一般）** の使用例。

| 同じコイン投げを書く2通り | 行き先 | 当てはまる定義 |
|---|---|---|
| $X(\omega) = H$ or $T$（ラベルそのまま） | $\lbrace H, T\rbrace$ | 定義1.6（一般） |
| $X(\omega) = 1$ or $0$（H=1, T=0 で符号化） | $\mathbb{R}$ | 定義1.7（実数値） |

本書があえて非数値で書いているのは、**「確率変数は数とは限らない」「抽象的な $\Omega$ に観測ラベルを取り付けるのが確率変数」** というメッセージを強調するため。実用的には数に符号化する方が多い。

## なぜこの分離が嬉しいのか

### 利点①: 抽象的な確率空間は1つ用意すれば使い回せる

具体的な問題（コイン、サイコロ、線分）ごとに別々の確率空間を構築するのではなく、**1つの大きな $\Omega$** の上に、複数の確率変数を載せられる。

### 利点②: 同じ確率空間上に複数の確率変数を自然に定義できる

例: 「コインの真偽 $Y$」と「コインの表裏 $X$」を同じ世界で同時に考える。

→ これが **結合分布、独立性、条件つき確率** の基盤になる（次節以降で本格化）。

### 利点③: 直感的なイメージ

> 世界はひとつあって、その中で色々な確率的な問題が同時に起きている（明日の天気、株価、コイン投げの結果、…）。
> 大きな $\Omega$ がそのすべての世界を代表し、 $\omega \in \Omega$ を一つ指定すれば確定的な世界がひとつ定まる。
> その世界の **何に注目するか** を確率変数 $X$ で取り出す。

### 具体例: コイン1回 + サイコロ1回

上の3つの利点を実感する例。

#### A. 別々の確率空間で考える場合

| | 確率空間 | 確率 |
|---|---|---|
| コイン | $\Omega_1 = \lbrace H, T\rbrace$ | $P_1(\lbrace H\rbrace ) = 1/2$ |
| サイコロ | $\Omega_2 = \lbrace 1, ..., 6\rbrace$ | $P_2(\lbrace i\rbrace ) = 1/6$ |

問題点: コインとサイコロは別々の世界に住んでいる。

- 「コインが表 **かつ** サイコロが6」を直接表現する事象が定義できない
- 結合確率 $P(\text{コイン}=H, \text{サイコロ}=6)$ を1つの確率空間として書けない
- 「独立だから掛ければよい」と暗黙に仮定するしかない

#### B. 1つの $\Omega$ にまとめる場合

- $\Omega = \lbrace H, T\rbrace  \times \lbrace 1, ..., 6\rbrace$（12個の元）
- $\mathcal{F} = 2^\Omega$
- $P(\lbrace (c, d)\rbrace ) = 1/12$

この $\Omega$ に **2つの確率変数** を載せる:
- $X : \Omega \to \lbrace H, T\rbrace$, $X(c, d) = c$
- $Y : \Omega \to \lbrace 1, ..., 6\rbrace$, $Y(c, d) = d$

これで全部 1つの確率空間上で書ける:

| 量 | 計算 |
|---|---|
| $P(X = H)$ | $P(\lbrace H\rbrace  \times \lbrace 1,...,6\rbrace ) = 6/12 = 1/2$ |
| $P(Y = 6)$ | $P(\lbrace H, T\rbrace  \times \lbrace 6\rbrace ) = 2/12 = 1/6$ |
| 結合確率 $P(X = H, Y = 6)$ | $P(\lbrace (H, 6)\rbrace ) = 1/12$ |
| 条件つき $P(X = H \mid Y = 6)$ | $\frac{1/12}{2/12} = 1/2$ |
| 独立性チェック | $1/12 = (1/2)(1/6)$ ✓ |

#### 何が嬉しいか

| | 別々の $\Omega$ | 1つの $\Omega$ + 複数の $X, Y$ |
|---|---|---|
| 結合確率 $P(X, Y)$ | ✗ 書けない | ✓ $\Omega$ 上の事象として定義可 |
| 条件つき $P(X \mid Y)$ | ✗ 別世界なので不明確 | ✓ 自然に書ける |
| 独立性のチェック | ✗ | ✓ 公式から検証できる |

→ **同じ $\Omega$ に複数の確率変数を住まわせる** ことで、結合分布・条件つき確率・独立性 が統一的に扱える。これが本節の哲学の実用的な御利益。

## $\sigma[X]$ — 確率変数から定まるσ加法族

ざっくり言うと **「 $X(\omega)$ の値だけから真偽が判定できる事象の集まり」**。

### 直感

サイコロ $\Omega = \lbrace 1, ..., 6\rbrace$ で確率変数 $X$ を「偶奇」とする:

$$
X(\omega) = \begin{cases} \text{偶} & (\omega \text{ が偶数}) \\\\ \text{奇} & (\omega \text{ が奇数}) \end{cases}
$$

$X(\omega)$ の値を見れば判定できる事象:
- 「 $X = \text{偶}$ か？」 → わかる
- 「 $X = \text{奇}$ か？」 → わかる

判定できない事象:
- 「 $\omega = 1$ か？」 → $X = 奇$ とわかっても、1/3/5 のどれか区別できない
- 「 $\omega \le 3$ か？」 → 同様

→ **$X$ の値だけで判定できる事象** だけを集めたのが $\sigma[X]$。

### 定義

$$
\sigma[X] := \lbrace X^{-1}(A) : A \in \mathcal{G}\rbrace 
$$

「値の空間 $S$ 側の可測集合 $A$ について、その逆像 $X^{-1}(A)$ を集めたもの」。

### 偶奇での計算

$X : \Omega \to \lbrace \text{偶}, \text{奇}\rbrace$, $\mathcal{G} = \lbrace \emptyset, \lbrace \text{偶}\rbrace , \lbrace \text{奇}\rbrace , S\rbrace$:

| $A$ | $X^{-1}(A)$ |
|---|---|
| $\emptyset$ | $\emptyset$ |
| $\lbrace \text{偶}\rbrace$ | $\lbrace 2, 4, 6\rbrace$ |
| $\lbrace \text{奇}\rbrace$ | $\lbrace 1, 3, 5\rbrace$ |
| $\lbrace \text{偶}, \text{奇}\rbrace$ | $\Omega$ |

よって $\sigma[X] = \lbrace \emptyset, \lbrace 2, 4, 6\rbrace , \lbrace 1, 3, 5\rbrace , \Omega\rbrace$（4個）。

サイコロの $2^\Omega$（64個）より小さい。 $X$ から得られる情報が「偶奇だけ」だから、 $\lbrace 1\rbrace$ のような細かい事象は入らない。

### 情報量としての $\sigma[X]$

| $X$ の情報量 | $\sigma[X]$ |
|---|---|
| 全部わかる（ $X(\omega) = \omega$ 恒等写像） | $2^\Omega$ 全体 |
| 偶奇だけわかる | $\lbrace \emptyset, \lbrace 2,4,6\rbrace , \lbrace 1,3,5\rbrace , \Omega\rbrace$ |
| 何もわからない（ $X(\omega) = c$ 定数） | $\lbrace \emptyset, \Omega\rbrace$ |

→ $\sigma[X]$ は「 $X$ を観測することで得られる情報量」を表すσ加法族。

### ベイズ推定では深入りしない

応用では条件つき期待値 $E[Y \mid \sigma[X]]$ などで使われるが、本書および本節では「**$X$ から決まる情報の量を表すσ加法族がある**」と認識すれば十分（著者も「あまり深入りしない」と書いている）。

## 注意 1.8: 技術的な警告

複数の確率変数を **同じ確率空間上に** 望む性質を持って構成できるか、という問題は実は非自明。

特にベイズ推定で重要な **「独立同分布な無限列」** を構成するには、 $\Omega$ や $\mathcal{F}$ を慎重に選ぶ必要がある。

> 結論: 自然に考えられる大抵の場合には可能。 $\Omega = [0, 1]$ 上に作れることもわかっている。
> 厳密な証明が知りたい人は専門的な確率論を勉強する必要がある、と著者は警告している。

→ 入門レベルでは **「事実として認めて先に進む」** で OK。

## 1.2.6 のまとめ

| 概念 | 役割 |
|---|---|
| 確率空間 $(\Omega, \mathcal{F}, P)$ | 抽象的な「ありうる世界」 |
| 確率変数 $X$ | 具体的な問題を設定する装置 |
| $\sigma[X]$ | $X$ の値から区別できる情報の量 |

#### 実務的な姿勢

ベイズ推定の理解だけが目的なら:
- 「 $P(X \in A)$ という記号の背後に、確率空間と可測な写像の仕組みがある」
- 「複数の確率変数は同じ $\Omega$ 上に住んでいる」

くらいの感覚で先に進めば十分。深い理論は必要に応じて立ち返る。
